Object Oriented Design

 

Many companies (looking at you Amazon) have an entire interview dedicated to object oriented design. 

 

The key to answering object oriented design questions is to recoginize that object oriented systems communicate in many ways similar to how we humans communicate in real life. 

Be sure to make yourself familiar with basic object oriented design concepts like inheritance, polymorphism, and a few design patterns.

The key steps to answer an object oriented design question are

  1. Ask clarifying questions, and identify use cases. 
  2. Identify major entities and their behaviors
  3. Identify subtypes of these entities and their behaviors
  4. Map the relation b/w these entities (assosciation). 
  5. Use design patterns to solve common problems, such as Singleton to prevent a class from being instantiated multiple times.
