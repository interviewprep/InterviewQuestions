Recursion

 

Recursion basically solves a larger problem by solving similar but smaller subproblems. 

 

In programming interviews, we use recursion when we know that we can solve the question at hand if somehow smaller subproblems are solved. For example, if you have to find the depth of a tree, we know that we can trivially calculate depth of a tree if we know the depth of the left subtree and right subtree. 

Most problems involving trees can be solved if their subproblems are solved, so they involve recursion.

Other reasons to use recursion is when a problem involves backtracking (you have to go down a path, back up and go down another path). 

 

Estimating time complexity of recursion

Don't forget to draw the recursion tree. Time complexity is the number of nodes in the tree, and space complexity is the maximum depth of the tree.

 

What is the time  & space  complexity of the following code ?

int fun (int n) {

if (n == 0) return 0;

return fun(n-1);

}

 

What is the time  & space  complexity of the following code ?

void fun (int n) {

if (n == 0) return ;

for (i = 0; i< n ; i++ ) {

fun(n-1);

}

}
