Arrays

 

Arrays are a very common topic for programming interviews, especially phone interviews.

 

Many array based interview questions are based on a few fundamental properties of arrays, such as ability to randomly access any element in constant time, and the inability to delete elements in the middle of an array, unlike a linked list.  

 

One very common interview question for entry level software engineering positions is the difference between linked lists and arrays. 

 

Operation Time complexity
Insert

O(1)

In dynamic arrays, if the array is full, a new array twice the size is created and all elements are copied over. The average “amortized” time complexity is still O(1). The doubling of the array every time keeps resizing operations very infrequent, keeping the amortized complexity constant.
Modify / Read O(1)
Delete There is no real way to delete one element.

 

 

Common techniques to solve array questions

Keep track of the value  you have seen so far. Sometimes you can save yourself traversing the array back and forth by just keeping of track of some values.

Swap numbers to the end of the array to “delete”. You cannot delete a random index in an array, but you can swap it with the last element and pretend to have deleted it.

Two pointer traversal.  Traversing an array with two pointers will give you some unique capabilities. The pointers may be located at opposite ends of an array, or at the same side of the array. Moving two pointers on the same side of the array, often called as the window technique because of the window formed between the two pointers is a very popular technique to solve many problems that need to find a subarray or a substring

 

Basic Questions

Count the largest possible sum of a subarray of size 3? https://github.com/interviewprep/InterviewQuestions/blob/master/arrays/SubArrayofSize3.java

Given an array of Integers, find the minimum distance between the given pair of numbers. For example in the array {1,3,6,2,7,9,8,2}, the minimum distance between (3,9) is 4 because 3 is at index 1 and 9 is at index 5. Likewise the minimum distance between (2,8) is 1 because 8 is at index 6 and 2 is at index 7 (it is also at index 3 but we want the minimum distance).  https://github.com/interviewprep/InterviewQuestions/blob/master/arrays/MinimumDistanceBetweenPair.java

Replace an array of numbers by the greatest number on the right hand side. Example: I/P: {2,6,7,4,3,2} O/P: {7,7,4,3,2,2}. Explanation: The largest element to the right hand side of index 0 is 7, so we place 7 at index 0, 2 is the largest number to the right of index 1 and so on. https://github.com/interviewprep/InterviewQuestions/blob/master/arrays/ReplaceWithGreatestNumberOnRigtSide.java

Given a sorted array of Integers, can you find if there are two numbers that add up to a given target?

How about 3 numbers?

How about finding the closest possible sum of two numbers to target? https://github.com/interviewprep/InterviewQuestions/blob/master/arrays/TwoSum.java
